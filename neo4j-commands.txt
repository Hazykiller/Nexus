# Nexus Social Network - Neo4j Cypher Commands Repository
Below are the primary Cypher commands utilized by the Next.js application to interact with the Neo4j database graph.

## 1. User Management
**Register User**
```cypher
CREATE (u:User {
  id: $id, email: $email, username: $username, name: $name,
  password: $password, avatar: '', coverPhoto: '', bio: '',
  website: '', location: '', dob: '', privacy: 'public',
  verified: false, createdAt: datetime(), updatedAt: datetime()
})
```

**Find User by Username or ID**
```cypher
MATCH (u:User {id: $userId}) RETURN u
// OR
MATCH (u:User {username: $username}) RETURN u
```

## 2. Social Connections
**Follow a User**
```cypher
MATCH (me:User {id: $userId}), (target:User {id: $targetId})
MERGE (me)-[:FOLLOWS]->(target)
```

**Unfollow a User**
```cypher
MATCH (me:User {id: $userId})-[r:FOLLOWS]->(target:User {id: $targetId})
DELETE r
```

**Get Follower Recommendations (Friends of Friends Algorithm)**
```cypher
MATCH (me:User {id: $userId})-[:FOLLOWS]->(friend:User)-[:FOLLOWS]->(suggestion:User)
WHERE NOT (me)-[:FOLLOWS]->(suggestion) AND me <> suggestion
WITH suggestion, COUNT(friend) as commonFriends
ORDER BY commonFriends DESC
RETURN suggestion, commonFriends LIMIT 10
```

## 3. Post Management
**Create a Post**
```cypher
MATCH (u:User {id: $userId})
CREATE (u)-[:CREATED]->(p:Post {
  id: $postId, content: $content, images: $images, video: $video,
  location: $location, visibility: $visibility,
  createdAt: datetime(), updatedAt: datetime()
})
```

**Like a Post**
```cypher
MATCH (u:User {id: $userId}), (p:Post {id: $postId})
MERGE (u)-[r:LIKES]->(p)
ON CREATE SET r.createdAt = datetime()
RETURN p
```

**Remove Like**
```cypher
MATCH (u:User {id: $userId})-[r:LIKES]->(p:Post {id: $postId})
DELETE r
```

## 4. Feed & Timeline
**Retrieve Home Feed (Posts from Followed Users & Self)**
```cypher
MATCH (me:User {id: $userId})
OPTIONAL MATCH (me)-[:FOLLOWS]->(followed:User)
WITH me, COLLECT(DISTINCT followed.id) AS followedIds
MATCH (author:User)-[:CREATED]->(p:Post)
WHERE (
  p.visibility = 'public'
  OR author.id IN followedIds
  OR author.id = $userId
)
OPTIONAL MATCH (p)<-[likes:LIKES]-()
OPTIONAL MATCH (me)-[myLike:LIKES]->(p)
RETURN p, author, COUNT(DISTINCT likes) AS likeCount, myLike IS NOT NULL AS isLiked
ORDER BY p.createdAt DESC LIMIT 10
```

## 5. Admin Visualization
**Extract Full Graph (Nodes & Relationships) for Dashboard**
```cypher
MATCH (n)
OPTIONAL MATCH (n)-[r]->(m)
RETURN collect(distinct n) as nodes, collect(distinct r) as edges
```
